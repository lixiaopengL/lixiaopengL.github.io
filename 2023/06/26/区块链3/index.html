数字货币

  • 直接通过非对称体系(公私钥)进行交易，不经过区块链，是有风险的。原因是:数字货币是可以复制的，因为他是文件。（花两次攻击double spending attack）

  • 中心化的方案，每一次交易都要进行确认，太麻烦。

  • 谁来发行货币？以后讲

  • 维护一个数据结构，区块链。

  • 为什么要记录来源？1、记录不是凭空捏造的。2、防止double spending attack。

  • B要知道A的公钥（通过 ），

  • A（10）会有一个公钥的哈希，A进行转账的时候，首先要确定，A的公钥与前面的哈希是对的上的。（这样就不会出现，BB拿自己的公钥冒充A的公钥，然后跟B说这是A的公钥，签名因为是BB的，公钥也是BB的，能通过验证。这样BB就偷了A的钱）。

  • 过程是一段脚本，A（2）交易的输入脚本和A（1）的输出脚本拼接起来，如果能运行，如果没有错误，就能交易成功。

  • BiCoin Script

  • 上面是Block header，只取上面的哈希，因为Merkle root hash的存在，就不存在被修改的可能性。

  • full node （全节点）（fully validating node）：参与区块链的构成。

  • light node(轻节点)：利用区块链的信息进行查询，不参与区块链的构成，计算机大多数都是轻节点，全节点的个数很少

  • 区块链是去中心化的账本，账本的内容要取得分布式的共识，distributed consensus

  • distributed hash table

  • FLP impossiblity result 异步系统里asynchronous——网络时延没有上限。

  • CAP Theorem-Consistency Availability，Partition tolerance。三个属性最多同时存在两个。

  • 达成Paxos，一定有Consistency（一致性）。

    Consensus in BitCoin （比特币共识机制）

  • 投票的方式，超过半数的节点认为没有问题。（membership,谁可以投票的问题？如果都是大公司投票，是可以的。但是，账户只有公钥私钥对别人怎末知道，所以说，如果恶意节点是一台超级计算机，什么事情都不干，就仅仅创建账户，只要达到半数以上，就能有控制权。肯定不行——————sybil attack 替身攻击）

  • H(block header)<=target

  • n Bits域（target目标阈值的编码）是否 符合比特币规定的难度系数要求。nonce是不是等于block header的哈希————是否有能力发布区块，是否有记账权。

  • 解释一下nonce：在区块链中，Nonce是一个随机生成的一次性数字，被用来保证区块的唯一性和安全性。区块链中使用Nonce的目的是确保区块链网络中的所有矿工都在尝试寻找生成正确的区块时做出了不同的尝试。它们都拥有相同的交易记录，但每个矿工都会使用不同的Nonce来证明自己已经找到了适当的解决方案。这是通过使用哈希函数对区块进行加密来实现的，该哈希函数要求包含一个非常难以猜测的Nonce值，从而使得区块的哈希值难以被篡改。当矿工找到一个正确的Nonce并生成一个有效的区块时，该区块被广播到网络的其他节点，然后加入到整个区块链中。通过使用Nonce验证，区块链可以避免攻击者对区块链进行篡改，并保证交易的安全性和完整性。在这种情况下，Nonce在保证区块链安全性方面起着非常关键的作用。

  • 不在最长合法链上的区块不予接受——分叉攻击（forking attack）
    回滚交易，A->B,分叉A->A,钱又发给了自己。

  • 两个矿工（节点）同时发布区块的情况——等长分叉。会有一个分叉最后胜出，输的叫做orphan block（孤儿块）。

  • 争夺记账权——（出块儿奖励）铸币权（coin base transaction）,凭空造币。初期可以造50个BTC。21万之后，减半，现在12.5BTC。

  • 算力投票（hash rate）：每秒钟试多少个nonce。由于puzzle friendly的性质，表示成功路上没有捷径（只能一个一个的试nonce）。

  • 解决女巫攻击（替身攻击），

  • 争夺记账权的过程叫做挖矿，比特币叫数字黄金，寻找nonce，即为淘金，获得记账权的节点叫做矿工。
